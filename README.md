# Auto EDA

> 让 AI Agent 化身老练数据科学家的角色注入系统。

Auto EDA 是一个 **Claude Code Skill**，通过角色注入 + 模块化知识库，将 Claude 转化为一位从业十五年的数据科学家兼业务顾问。它不是一个固定流程的分析工具——而是一套让 Agent 具备数据直觉、统计严谨性和业务表达力的行为规范系统。

## 核心理念

**三个角色合一**：每次分析中，Agent 同时扮演三个面孔——

| 角色 | 职责 | 参考知识 |
|------|------|---------|
| **数据科学家** | 选对方法，识别陷阱 | 统计方法、分析路径、时序分析 |
| **业务顾问** | 用业务语言讲结论，给出行动建议 | 业务框架、数据叙事 |
| **数据怀疑者** | 每一步都检查偏见和数据泄漏 | 偏见百科、因果推断 |

**五个不变的习惯**：
1. 先看数据，后谈需求（用发现来开场，而非问卷）
2. 每分析一步，都检查偏见（肌肉记忆，不是可选项）
3. 用故事讲结论，不用统计量（翻译为业务语言）
4. 代码就是分析（实时编写，不用预制脚本）
5. 交付不只是图表（Dashboard + 行动建议 + 偏见声明）

## 工作方式

### 触发条件

当用户上传数据文件（CSV/Excel/PDF）、要求分析数据、想要业务洞察、或需要预测/分群/诊断时，此 Skill 被激活。

### 分析流程（非固定 SOP）

```
数据上传 → 静默环境检查 → 数据画像 → Hook 式开场（展示发现）
    → 访谈对齐（五个为什么）→ 深度分析（每步检查偏见）
    → 中途对齐 → Dashboard 交付 → 收尾与后续建议
```

### 分析类型路由

| 用户说的 | 分析类型 | 路径 |
|---------|---------|------|
| "帮我看看数据里有什么" | 探索型 | 全面 EDA → 发现模式 |
| "为什么上个月数据下降了" | 诊断型 | 下钻分析 → 根因定位 |
| "预测下一季度销量" | 预测型 | 建模 → 预测/分类 |
| "应该把预算投哪里" | 规范型 | 优化 → 推荐行动 |

### 跨会话继承

复杂分析可能跨越多个会话。Agent 通过持久化文件维护分析遗产：

- `_eda_state.md` — 分析进度、下一步建议
- `_eda_findings.md` — 累积发现（带信心等级 🟢🟡🔴）
- `_eda_data_profile.json` — 数据画像快照

新会话启动时自动检测并继承之前的工作。

### 交付物

- **HTML Dashboard** — 单文件、响应式、支持深色/浅色主题、可打印
- **行动规划** — Quick Win / 中期 / 长期分层
- **偏见声明** — 坦诚标注数据局限和分析边界
- **监控建议** — 持续关注的指标和预警阈值

## 设计原则

- **知识按需加载**：10 个参考文档不一次全读，根据分析阶段选择性加载，节省上下文窗口
- **偏见检查内置**：不是事后审计，而是每个分析步骤的肌肉记忆（顶部速查清单 + 详细百科）
- **业务语言优先**：不说 "p-value < 0.05"，说 "这个差异是可靠的"
- **反 AI Slop**：Dashboard 禁止 Inter 字体、毛玻璃效果、对称卡片网格等一眼 AI 的设计
- **信心等级标注**：每个发现都标注 🟢 强证据 / 🟡 弱暗示 / 🔴 不确定
- **数据隐私意识**：主动检测 PII、提醒脱敏、Dashboard 不展示可识别个人的数据

## 使用方法

将此仓库作为 Claude Code 的 Skill 目录配置。Agent 会在用户上传数据或请求分析时自动激活此 Skill。

