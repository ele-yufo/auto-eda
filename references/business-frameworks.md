# 业务分析框架库

> 数据分析不在真空中进行。这些框架帮助 Agent 用业务思维组织分析，而不只是跑统计。

---

## 一、客户分析框架

### 1.1 RFM 分析

| 维度 | 含义 | 计算 |
|------|------|------|
| **R** (Recency) | 最近一次交易距今多久 | `今天 - 最后交易日期` |
| **F** (Frequency) | 交易频率 | `总交易次数 / 时间窗口` |
| **M** (Monetary) | 交易金额 | `总消费金额` |

**分群方法**：每个维度分 3-5 档 → 组合得到客户群 →
- RFM 高-高-高 → VIP 客户（维护）
- RFM 低-高-高 → 即将流失的高价值客户（紧急挽回）
- RFM 高-低-低 → 新客（培育）

### 1.2 客户生命周期价值 (CLV/LTV)

```
简化公式：LTV = ARPU × 平均生命周期 × 毛利率
其中：
  ARPU = 总收入 / 用户数
  平均生命周期 = 1 / 流失率
```

**业务价值**：
- LTV / CAC > 3 → 健康的获客模型
- LTV / CAC < 1 → 亏钱获客，需要调整

### 1.3 客户分群（非 RFM）

| 分群维度 | 适用场景 |
|---------|---------|
| 行为分群 | 根据使用行为（活跃度、功能使用、路径） |
| 价值分群 | 根据收入贡献 |
| 需求分群 | 根据产品偏好 |
| 人口学分群 | 根据年龄/地区/职业（通常和上面组合使用） |

## 二、增长分析框架

### 2.1 AARRR 海盗指标

| 阶段 | 关键指标 | 典型分析 |
|------|---------|---------|
| **A**cquisition（获客） | 注册量、渠道转化率 | 渠道对比、CAC 分析 |
| **A**ctivation（激活） | 首次关键行为完成率 | 激活漏斗、Aha Moment 识别 |
| **R**etention（留存） | N 日留存率、DAU/MAU | 留存曲线、群体分析 |
| **R**evenue（收入） | ARPU、LTV、转化率 | 付费漏斗、定价分析 |
| **R**eferral（推荐） | K 因子、NPS | 病毒传播分析 |

### 2.2 漏斗分析

```
步骤 1: 访问页面 ──→ 100% (基准)
步骤 2: 注册      ──→  25% (75% 流失)  ← 流失最大的环节
步骤 3: 首次使用  ──→  18% (28% 流失)
步骤 4: 付费      ──→   5% (72% 流失)

关注点：
- 哪一步流失最大？→ 优先优化
- 流失的用户有什么特征？→ 分群分析
- 不同渠道的漏斗差异？→ 渠道质量评估
```

## 三、实验分析框架

### 3.1 A/B 测试结果分析

```
检查清单：
□ 样本量是否足够？（功效分析）
□ 随机化是否成功？（对照组 vs 实验组的基线特征是否一致）
□ 是否有 SRM (Sample Ratio Mismatch)？
□ p-value + 置信区间 + 效应量（三者都要报告）
□ 是否有 novelty effect（新鲜感效应）？
□ 分人群看效果是否一致？（异质性分析）
□ 对长期指标的影响？（不只看短期转化）
```

### 3.2 A/B 测试设计建议

| 要素 | 建议 |
|------|------|
| 样本量计算 | 基于 MDE（最小可检测效应）和 Baseline 转化率 |
| 运行时长 | 至少覆盖 1 个完整业务周期（通常 1-2 周） |
| 成功指标 | 1 个主指标 + 2-3 个辅助指标 + 1 个护栏指标 |
| 注意事项 | 不要早停、不要 peek（窥探 p-value） |

## 四、财务分析框架

### 4.1 常用财务指标

| 指标 | 公式 | 健康值参考 |
|------|------|-----------|
| ARPU | 收入 / 用户数 | 行业差异大 |
| CAC | 获客总成本 / 新客户数 | LTV/CAC > 3 |
| LTV | ARPU × 生命周期 × 毛利率 | > 3×CAC |
| 毛利率 | (收入 - COGS) / 收入 | > 50% (SaaS) |
| 流失率 | 流失用户数 / 期初用户数 | < 5%/月 |
| MRR 增长率 | (本月 MRR - 上月 MRR) / 上月 MRR | > 10%/月 (早期) |

### 4.2 单位经济学分析

```
单位收入 = 客单价 × 复购率 × 生命周期
单位成本 = CAC + 服务成本/客户
单位利润 = 单位收入 - 单位成本

如果 单位利润 > 0 → 模式可持续
如果 单位利润 < 0 → 需要优化哪个环节？
```

## 五、归因模型

| 模型 | 原理 | 适用场景 | 偏见风险 |
|------|------|---------|---------|
| 首次归因 | 功劳归第一个触点 | 评估品牌曝光效果 | 低估后续触点 |
| 末次归因 | 功劳归最后一个触点 | 评估转化驱动力 | 低估前期触点 |
| 线性归因 | 功劳平均分配 | 无先验判断时 | 过于简化 |
| 时间衰减 | 离转化越近权重越高 | 销售周期长 | 仍有主观判断 |
| Shapley 值 | 按边际贡献分配 | 最公平但计算量大 | 计算假设 |

**Agent 行为**：有归因分析需求时，默认用线性归因作为基线，然后展示首次和末次的对比，让用户理解差异。

## 六、价格弹性分析

### 6.1 价格弹性的定义

```
价格弹性 = 需求量变化% / 价格变化%
         = (ΔQ/Q) / (ΔP/P)

解读：
  |弹性| > 1  → 弹性需求（价格敏感，降价能带来更多收入）
  |弹性| < 1  → 刚性需求（价格不敏感，涨价通常增加收入）
  |弹性| ≈ 1  → 单位弹性（价格变动对收入影响不大）
```

### 6.2 估计方法

**方法 1：历史数据回归（最常用）**

```python
import numpy as np
from sklearn.linear_model import LinearRegression

# 对数-对数模型：ln(Q) = α + β × ln(P) + ε
# β 即为价格弹性
df['ln_quantity'] = np.log1p(df['quantity'])
df['ln_price'] = np.log1p(df['price'])

model = LinearRegression()
model.fit(df[['ln_price']], df['ln_quantity'])
elasticity = model.coef_[0]  # β 就是弹性系数
```

⚠️ **必须控制混杂变量**：季节性、促销活动、竞品价格等都会影响需求。加入控制变量后的弹性估计更可靠。

**方法 2：分组前后对比（有价格变动事件时）**

```
弹性 ≈ (变动后销量 - 变动前销量) / 变动前销量
      ÷ (变动后价格 - 变动前价格) / 变动前价格
```

⚠️ 这是最简单的弧弹性估计，容易被其他因素干扰。优先使用回归方法。

### 6.3 业务应用

| 弹性值 | 定价建议 |
|--------|---------|
| 高弹性 (> 1.5) | 降价促销有效，关注销量增长 |
| 中弹性 (0.8-1.5) | 价格调整需谨慎测试 |
| 低弹性 (< 0.8) | 有涨价空间，关注利润率 |

**Agent 行为**：
- 估计弹性时必须检查价格变动是否与其他因素（促销、季节、竞品）同时发生
- 分客户群计算弹性（不同群体的价格敏感度可能差异很大）
- 用置信区间而非单点估计报告弹性值
- 提醒用户：历史弹性不代表未来弹性，建议用 A/B 测试验证
