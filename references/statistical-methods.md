# 统计方法速查 & 因果推断

> 何时用什么检验？如何从相关性走向因果性？

---

## 一、假设检验选择流程图

```
比较两组或多组？
├── 比较两组
│   ├── 数据类型？
│   │   ├── 连续型
│   │   │   ├── 数据正态？（Shapiro-Wilk p > 0.05）
│   │   │   │   ├── YES → 样本关系？
│   │   │   │   │   ├── 独立样本 → 独立样本 t-test
│   │   │   │   │   └── 配对样本 → 配对样本 t-test
│   │   │   │   └── NO → 样本关系？
│   │   │   │       ├── 独立样本 → Mann-Whitney U
│   │   │   │       └── 配对样本 → Wilcoxon signed-rank
│   │   │   └── 方差齐？（Levene p > 0.05）
│   │   │       ├── YES → 标准 t-test
│   │   │       └── NO → Welch's t-test
│   │   └── 分类型
│   │       └── 卡方检验 (Chi-square) / Fisher 精确检验（小样本）
│   │
│   └── 比较多组
│       ├── 连续型
│       │   ├── 正态 + 方差齐 → 单因素 ANOVA
│       │   └── 非正态 → Kruskal-Wallis
│       └── 分类型
│           └── 卡方检验
│
└── 事后多重比较
    ├── ANOVA 显著后 → Tukey HSD / Bonferroni
    └── Kruskal-Wallis 显著后 → Dunn's test
```

## 二、相关分析

| 方法 | 适用条件 | 输出 |
|------|---------|------|
| Pearson | 两个连续变量，线性关系，正态分布 | r ∈ [-1, 1] |
| Spearman | 两个连续/有序变量，单调关系，不要求正态 | ρ ∈ [-1, 1] |
| Kendall | 类似 Spearman，对异常值更稳健，小样本更可靠 | τ ∈ [-1, 1] |
| 点二列相关 | 一个连续 + 一个二分类 | r_pb |
| Cramér's V | 两个分类变量 | V ∈ [0, 1] |

**相关性强度解读**：
| |r| 范围 | 强度 |
|---|---|
| 0.00 - 0.10 | 无相关 |
| 0.10 - 0.30 | 弱相关 |
| 0.30 - 0.50 | 中等相关 |
| 0.50 - 0.70 | 强相关 |
| 0.70 - 1.00 | 极强相关 |

## 三、效应量

> p-value 告诉你"差异是否存在"，效应量告诉你"差异有多大"。**两者都要报告。**

| 效应量 | 适用场景 | 小 | 中 | 大 |
|--------|---------|---|---|---|
| Cohen's d | 两组均值比较 | 0.2 | 0.5 | 0.8 |
| η² (Eta squared) | ANOVA | 0.01 | 0.06 | 0.14 |
| Cramér's V | 分类×分类 | 0.1 | 0.3 | 0.5 |
| R² | 回归 | 0.02 | 0.13 | 0.26 |
| OR (Odds Ratio) | 二分类 | 1.5 | 2.5 | 4.3 |

**业务语言翻译**：
- Cohen's d = 0.2 → "两组有差异，但差异较小，实际意义有限"
- Cohen's d = 0.8 → "两组差异显著且实质性的，值得关注"
- R² = 0.13 → "模型能解释约 13% 的变化，说明还有其他重要因素没被捕获"

## 四、样本量与统计功效

**统计功效** = 在真实差异存在时，能检测到它的概率。

- 推荐功效 ≥ 0.80（即 80% 的把握检测到真实差异）
- 功效受三个因素影响：效应量、样本量、显著性水平

**经验法则**（中等效应量，α=0.05，Power=0.80）：
| 检验类型 | 每组最少样本量 |
|---------|-------------|
| 独立 t-test | ~64 |
| 配对 t-test | ~34 |
| 单因素 ANOVA (3 组) | ~52/组 |
| 卡方 (2×2) | ~87 |
| 相关 (r=0.3) | ~85 |

**Agent 行为**：
- 样本量 < 30：标注"结果可能不可靠"
- 接近最小样本量：标注"证据边缘，需谨慎解读"

## 五、参数 vs 非参数检验

| 条件 | 参数检验 | 非参数检验 |
|------|---------|----------|
| 正态分布 | ✅ 适用 | ✅ 适用（但功效略低） |
| 非正态分布 | ❌ 不适用 | ✅ 适用 |
| 小样本 (< 30) | ⚠️ 需严格检验正态性 | ✅ 更安全 |
| 有序数据 | ❌ | ✅ |
| 有极端异常值 | ❌ 被严重影响 | ✅ 更稳健 |

**经验法则**：如果不确定，先做正态性检验（Shapiro-Wilk），p < 0.05 则用非参数。

## 六、因果推断 — 从相关性往前走一步

> 这是 Auto EDA 区别于传统 EDA 的关键能力。当发现重要的相关性时，不止于说"相关≠因果"，而是引导用户思考因果关系。

### 6.1 DAG 因果图思维

**什么是 DAG**：用有向图表示变量间的因果关系假设。

```
举例：收入 → 消费
           ↑
         教育水平 → 收入
                    ↑
                  家庭背景

如果我们想分析"教育对消费的影响"，需要控制"家庭背景"，
但不应该控制"收入"（因为收入是中介变量，中介）。
```

**Agent 行为**：
- 发现重要相关性时，画出假设的因果链（用文字描述）
- 识别出哪些是混杂变量（都需要控制）、中介变量（不应控制）、碰撞变量（控制后反而引入偏差）
- 邀请用户基于业务知识确认或修正

### 6.2 准实验方法

**当无法做 RCT（随机对照实验）时，可以考虑的方法**：

| 方法 | 适用条件 | 核心假设 |
|------|---------|---------|
| **Difference-in-Differences (DiD)** | 有处理组和对照组的前后数据 | 平行趋势假设 |
| **倾向得分匹配 (PSM)** | 有处理组和对照组，但非随机分配 | 可观测变量能解释分组差异 |
| **断点回归 (RDD)** | 有明确的阈值/截止点 | 阈值附近的个体近似随机 |
| **工具变量 (IV)** | 有影响处理但不直接影响结果的变量 | 排斥性限制 |

**Agent 执行边界**——明确什么该做、什么不该做：

| Agent 应该做 | Agent 不应该做 |
|---|---|
| 识别数据中存在因果性问题的信号 | 声称从观察数据中"证明了"因果关系 |
| 用文字画出假设因果链（DAG），邀请用户确认 | 未经用户确认就按假设因果链做分析 |
| 如果数据有处理前后的对照结构 → 尝试 DiD | 在假设未验证时完整执行 PSM/RDD/IV |
| 告诉用户"如果想确认因果关系，建议做 A/B 测试" | 将准实验方法的结果当作确定因果结论 |
| 运行 6.3 相关到因果的检查清单 | 跳过检查清单直接给因果性判断 |

**简单判断**：如果用户没有明确的因果推断需求，Agent 做到"画出因果假设 + 跑完检查清单 + 给出建议"就够了。完整的准实验方法执行需要用户确认且数据结构支持。

### 6.3 从相关到因果的检查清单

发现 A 和 B 相关后，逐步检查：

```
□ 1. 方向：A→B 还是 B→A？（时间顺序、领域知识）
□ 2. 混杂：有没有 C 同时影响 A 和 B？
□ 3. 控制混杂后，相关性还存在吗？
□ 4. 剂量效应：A 越强，B 越强吗？（梯度关系）
□ 5. 一致性：在不同子组中都观察到这个关系吗？
□ 6. 机制：有合理的理论解释吗？
□ 7. 可操纵性：改变 A，能影响 B 吗？（这通常需要实验验证）
```

## 七、模型解释 — SHAP 指引

> 建模后，特征重要性告诉你"哪些因素重要"，SHAP 告诉你"每个因素对每个预测的具体贡献"。

### 7.1 何时使用 SHAP

| 场景 | 是否用 SHAP |
|------|-----------|
| 树模型（RF/XGBoost/LightGBM）已训练 | ✅ 推荐，TreeExplainer 很快 |
| 线性模型 | 可选（系数本身就可解释） |
| 探索性 EDA，未建模 | ❌ 不需要 |
| 用户需要解释"为什么模型做出这个预测" | ✅ 必须 |

### 7.2 标准用法

```python
import shap

# 树模型（推荐，速度快）
explainer = shap.TreeExplainer(model)
shap_values = explainer(X_test)

# 三张关键图
shap.plots.bar(shap_values)           # 全局特征重要性
shap.plots.beeswarm(shap_values)      # 每个特征的影响方向和分布
shap.plots.waterfall(shap_values[0])  # 单个样本的预测解释
```

### 7.3 业务语言翻译

| SHAP 输出 | 业务语言 |
|-----------|---------|
| 特征 X 的 SHAP 值 = +0.15 | "X 让这个客户的流失概率增加了 15 个百分点" |
| 全局排名第一的特征是"最近登录天数" | "影响客户是否流失的最关键因素是活跃度" |
| 蜂群图显示高值红点集中在正侧 | "这个因素越高，流失风险越大" |

### 7.4 注意事项

- ⚠️ SHAP 解释的是**模型学到的关系**，不是因果关系
- ⚠️ 如果模型本身有数据泄漏，SHAP 解释也会误导
- ⚠️ 高度相关的特征会分散 SHAP 值——先检查多重共线性
- 在 Dashboard 中优先展示 bar 图（全局重要性），蜂群图放附录

## 八、Bootstrap 与置换检验

> 当数据不满足参数检验假设（非正态、小样本、分布未知）时，Bootstrap 和置换检验是强大的替代工具。

### 8.1 何时使用

| 场景 | 推荐方法 |
|------|---------|
| 估计统计量的置信区间（均值、中位数、比率） | Bootstrap |
| 小样本下检验两组差异 | 置换检验 |
| 不确定分布是否满足参数假设 | 两者皆可 |
| 需要向非技术受众解释"为什么这个结论可靠" | Bootstrap（直觉好理解） |

### 8.2 Bootstrap 置信区间

```python
import numpy as np

def bootstrap_ci(data, stat_func=np.mean, n_boot=10000, ci=95):
    """计算 Bootstrap 置信区间"""
    boot_stats = [stat_func(np.random.choice(data, size=len(data), replace=True))
                  for _ in range(n_boot)]
    lower = np.percentile(boot_stats, (100 - ci) / 2)
    upper = np.percentile(boot_stats, 100 - (100 - ci) / 2)
    return lower, upper

# 使用示例
ci_low, ci_high = bootstrap_ci(df['revenue'].dropna().values)
print(f"均值的 95% Bootstrap 置信区间: [{ci_low:.2f}, {ci_high:.2f}]")
```

**业务语言**："我们用 10000 次随机重采样来估计这个指标的可靠范围——真实值很可能在 [X, Y] 之间。"

### 8.3 置换检验

```python
def permutation_test(group_a, group_b, n_perm=10000):
    """检验两组差异是否显著"""
    observed_diff = np.mean(group_a) - np.mean(group_b)
    combined = np.concatenate([group_a, group_b])
    perm_diffs = []
    for _ in range(n_perm):
        np.random.shuffle(combined)
        perm_diffs.append(np.mean(combined[:len(group_a)]) - np.mean(combined[len(group_a):]))
    p_value = np.mean(np.abs(perm_diffs) >= np.abs(observed_diff))
    return observed_diff, p_value
```

**业务语言**："我们随机打乱分组 10000 次，看有多少次偶然产生的差异比实际差异更大。如果很少（< 5%），说明这个差异不太可能是偶然的。"

## 九、贝叶斯方法入门

> 当样本小或有合理的先验知识时，贝叶斯方法比频率方法更有用。

### 9.1 何时考虑贝叶斯

| 场景 | 推荐 |
|------|------|
| A/B 测试样本很小（< 100/组） | 贝叶斯 A/B 测试 |
| 有强先验（"历史转化率约 3%"） | 贝叶斯更新 |
| 需要说"A 比 B 好的概率是 X%" | 贝叶斯（频率方法无法给出这种结论） |
| 大样本 + 无先验信息 | 频率方法就够了 |

### 9.2 贝叶斯 A/B 测试（转化率）

```python
from scipy import stats

# Beta-Binomial 模型
# 先验：Beta(1, 1)（均匀先验，无偏向）
# 数据：A 组 50/1000 转化，B 组 65/1000 转化
a_alpha, a_beta = 1 + 50, 1 + (1000 - 50)   # A 的后验
b_alpha, b_beta = 1 + 65, 1 + (1000 - 65)   # B 的后验

# 蒙特卡洛模拟：B 比 A 好的概率
a_samples = np.random.beta(a_alpha, a_beta, 100000)
b_samples = np.random.beta(b_alpha, b_beta, 100000)
prob_b_better = np.mean(b_samples > a_samples)
print(f"B 方案优于 A 的概率: {prob_b_better:.1%}")

# B 比 A 好多少
lift = (b_samples - a_samples) / a_samples
print(f"预期提升: {np.mean(lift):.1%} (90% 区间: [{np.percentile(lift, 5):.1%}, {np.percentile(lift, 95):.1%}])")
```

**业务语言**："根据目前的数据，B 方案优于 A 的概率是 92%。如果 B 确实更好，预期提升在 10%-40% 之间。"

### 9.3 Agent 行为

- 当样本量小且用户有先验知识时，主动提议贝叶斯方法
- 贝叶斯结果和频率结果可以同时报告——它们是互补的
- 用"概率"语言沟通，比 p-value 更直觉

## 十、生存分析

> 当分析"客户生命周期"、"设备故障时间"等"事件发生时间"的问题时使用。

### 10.1 何时使用

| 场景 | 方法 |
|------|------|
| "客户平均多久会流失？" | Kaplan-Meier 生存曲线 |
| "哪些因素影响客户留存时间？" | Cox 比例风险模型 |
| "不同方案的留存率有差异吗？" | Log-rank 检验 |

### 10.2 关键概念

```python
from lifelines import KaplanMeierFitter, CoxPHFitter

# Kaplan-Meier 生存曲线
kmf = KaplanMeierFitter()
kmf.fit(
    durations=df['tenure_days'],     # 持续时间
    event_observed=df['churned'],     # 事件是否发生（1=流失, 0=仍在）
)
kmf.plot_survival_function()
print(f"中位生存时间: {kmf.median_survival_time_:.0f} 天")

# Cox 回归（多因素）
cph = CoxPHFitter()
cph.fit(df[['tenure_days', 'churned', 'age', 'plan_type', 'usage']],
        duration_col='tenure_days', event_col='churned')
cph.print_summary()
```

**业务语言**：
- "50% 的客户在注册后 180 天内流失"（中位生存时间）
- "高级套餐用户的流失风险比基础套餐低 40%"（风险比 HR=0.6）

### 10.3 注意事项

- ⚠️ 生存分析最大的特点是能正确处理**右删失数据**——那些"还没流失但观察期结束"的客户。简单计算"流失率"会忽略这些客户
- ⚠️ 比较两组生存曲线用 log-rank 检验，不用 t-test
