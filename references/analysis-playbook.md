# 分析方法论 & 决策树

> 面对数据时，应该使用什么分析路径？这是 Agent 的分析思维导图。

---

## 一、分析目标分类

用户的分析需求通常可以归为四类：

| 类型 | 核心问题 | 分析方向 |
|------|---------|---------|
| **探索型** | "数据里有什么有趣的？" | 全面 EDA → 发现模式和异常 |
| **诊断型** | "为什么指标下降了？" | 下钻分析 → 根因定位 |
| **预测型** | "未来会怎样？" | 建模 → 预测/分类 |
| **规范型** | "我应该怎么做？" | 优化 → 推荐行动 |

---

## 二、分析路径决策树

```
用户给了数据 →
├── 目标变量明确？
│   ├── YES → 目标变量类型？
│   │   ├── 连续型 → 回归分析路径
│   │   ├── 二分类 → 分类分析路径
│   │   ├── 多分类 → 多分类路径
│   │   └── 时间序列 → 时序分析路径（参考 `references/timeseries-methods.md`）
│   └── NO → 探索型分析路径
│
├── 有分组/对比需求？
│   ├── 两组对比 → t-test / Mann-Whitney
│   ├── 多组对比 → ANOVA / Kruskal-Wallis
│   └── 前后对比 → 配对检验 / DiD
│
├── 有分群需求？
│   └── 聚类分析路径（K-Means / 层次聚类）
│
└── 有关联分析需求？
    └── 相关分析 → 因果推断检查点
```

## 三、标准分析套路（按场景）

### 场景 1：客户流失分析
```
1. 定义流失（与用户对齐：多久不活跃算流失？）
2. 描述性分析：流失率趋势、流失客户画像
3. 对比分析：流失 vs 留存客户在各维度的差异
4. ⚠️ 辛普森悖论检查（详见 `bias-and-pitfalls.md`）：分渠道/分产品看，结论是否一致？
5. 建模：预测流失概率 → 特征重要性 → SHAP 解释
6. ⚠️ 数据泄漏检查：特征是否包含流失后才有的信息？
7. 行动建议：高风险客户识别 + 干预策略
```

### 场景 2：定价策略分析
```
1. 价格分布分析：不同价位段的销量/收入分布
2. 价格弹性估计：价格变动 vs 销量变动
3. 分组定价分析：不同客户群对价格的敏感度差异
4. ⚠️ 因果检查：价格变化是否与其他变量同时发生？
5. 竞品对标（如有数据）
6. 建议：最优价格区间 + 差异化定价策略
```

### 场景 3：营销归因分析
```
1. 各渠道的转化漏斗
2. 首次/末次/线性归因对比
3. ⚠️ 自选择偏差检查：不同渠道的用户本身就不同吗？
4. 增量分析（如果有 A/B 测试数据）
5. 建议：各渠道投放 ROI 排名 + 预算分配建议
```

### 场景 4：运营指标异常诊断
```
1. 确认异常：指标变化是否超出正常波动？（Z-score / 控制图）
2. 时间定位：异常发生的精确时间点
3. 维度下钻：按地区/产品/渠道/用户群拆分，定位异常来源
4. ⚠️ 回归均值检查：这是异常还是正常回归？
5. 交叉参考：同时期有什么外部事件？（节假日、竞品动作、系统变更）
6. 根因假设 → 数据验证
```

### 场景 5：用户分群
```
1. 选择分群维度（行为 / 价值 / 人口学）
2. 特征工程：RFM / 行为频率 / 使用深度
3. 标准化 + 聚类（K-Means / DBSCAN）
4. 确定最优 K（肘部法则 + 轮廓系数）
5. 分群画像：每个群的特征描述（用业务语言）
6. ⚠️ 分群的稳定性检查：换个时间窗口，分群结果一致吗？
7. 行动建议：每个群的差异化策略
```

## 四、分析深度自适应

根据数据特征和用户需求自动调整：

| 信号 | 分析深度 | 预期时间 |
|------|---------|---------|
| 数据少（< 1000 行）+ 探索性 | 快速 EDA 报告 | 10 分钟 |
| 数据中等 + 明确对比需求 | 统计检验 + 可视化 | 20-30 分钟 |
| 数据大 + 明确预测目标 | 完整建模流程 | 30-60 分钟 |
| 时序数据 | 自动启用时序模块（参考 `timeseries-methods.md`） | 按复杂度 |
| 多表关联数据 | 先做 JOIN 策略 | 按规模 |

## 五、启发式规则

"当你看到 X 时，应该想到 Y"：

| 看到 | 应该想到 |
|------|---------|
| 偏度 > 2 | 对数变换？极端值驱动？ |
| 两列相关性 > 0.9 | 多重共线性 → 建模时考虑删除一个 |
| 分类变量有 50+ 个类别 | 高基数 → 需要编码策略 |
| 目标变量严重不平衡（> 10:1） | 需要 SMOTE / 调整阈值 / 用 PR-AUC 评估 |
| 某列 > 50% 缺失 | 考虑删除该列，或调查缺失原因 |
| 模型 F1 > 0.95 | 高度怀疑数据泄漏 |
| 特征重要性 top1 远超其他 | 可能是泄漏或代理变量 → 深入检查 |
| 训练/验证差距 > 10% | 过拟合信号 |
| 数据有时间列 | 考虑时序分析，按时间划分训练/测试 |
